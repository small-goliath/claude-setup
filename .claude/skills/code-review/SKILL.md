## 리뷰 기준

### 프로젝트 규칙

프로젝트 코드 작성 규칙:
```
!`cat $PROJECT_FORMAT_PATH`
```

코드 스타일 규칙:
```
!`cat $CHECKSTYLE_RULES_PATH`
```

### 검토 항목

#### 1. 프로젝트 코드 작성 규칙 준수
- 네이밍 컨벤션
- 패키지 구조
- 아키텍처 패턴
- 코드 포맷

#### 2. 로직 검증
- **시간 복잡도**: 알고리즘 효율성
- **공간 복잡도**: 메모리 사용량
- **가독성**: 코드 이해 용이성
- **유지보수성**: 변경 용이성
- **에러 처리**: 예외 상황 처리

#### 3. 보안 체크
- 입력 값 검증
- SQL Injection 방지
- XSS 방지
- 권한 체크
- 민감 정보 노출 방지

#### 4. 의존성 및 사이드 이펙트
- 의존성 주입 적절성
- 순환 의존성 확인
- 전역 상태 변경
- 부작용 최소화

#### 5. 사용자 지정 컨벤션
- Checkstyle 규칙 준수
- Formatter 규칙 준수
- 프로젝트별 특수 규칙

## 피드백 레벨

### CRITICAL (치명적)
- 보안 취약점
- 데이터 손실 가능성
- 시스템 장애 가능성
- **즉시 수정 필수**

### HIGH (높음)
- 성능 저하
- 메모리 누수
- 로직 오류
- **수정 강력 권장**

### MEDIUM (보통)
- 가독성 문제
- 코드 중복
- 규칙 위반
- **개선 권장**

### LOW (낮음)
- 스타일 문제
- 네이밍 개선
- 리팩토링 제안
- **선택적 개선**
