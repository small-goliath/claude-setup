## 설계 단계

### 1. 요구사항 분석
- Redmine 이슈 분석 결과 검토
- 기능 요구사항 파악
- 비기능 요구사항 파악

### 2. 현황 파악
- 영향받는 파일 식별 (Glob, Grep 활용)
- 현재 아키텍처 이해
- 관련 코드 리뷰 (Read 활용)

### 3. 설계
- 구현 방안 수립
- 클래스/메서드 설계
- 데이터베이스 스키마 변경 (필요시)
- API 인터페이스 설계 (필요시)

### 4. 테스트 전략
- 단위 테스트 계획
- 통합 테스트 계획
- E2E 테스트 계획 (필요시)

### 5. 사용자 확인
AskUserQuestion을 사용하여 설계 승인:
- 설계 접근법
- 예상 변경 파일
- 테스트 전략
